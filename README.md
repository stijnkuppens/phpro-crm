# PHPro CRM

Next.js 16 + Supabase (self-hosted) CRM application.

## Prerequisites

- [Node.js](https://nodejs.org/) 20+
- [Docker](https://www.docker.com/) & Docker Compose
- [Task](https://taskfile.dev/) (`brew install go-task`)

## Setup from Scratch

### 1. Clone the repository

```bash
git clone <repo-url> phpro-crm
cd phpro-crm
```

### 2. Run the setup wizard

```bash
task setup
```

This interactive script will:
- Check prerequisites (Docker, Node.js, npm)
- Prompt for project name, display name, and ports
- Copy `.env.example` to `.env` and generate all secrets (JWT, passwords, API keys)
- Install npm dependencies
- Start all Supabase Docker services
- Run database migrations
- Seed the database (production data + demo fixtures)
- Generate TypeScript types from the database schema

Once complete, you're ready to go:

```bash
task dev
```

### 3. Access the app

| Service | URL | Credentials |
|---------|-----|-------------|
| App | http://localhost:3000 | `admin@example.com` / `admin123456` |
| Supabase Studio | http://localhost:3001 | `admin` / `admin` |
| Supabase API (Kong) | http://localhost:8000 | — |

### Manual setup (if not using the wizard)

```bash
# 1. Copy and configure environment
cp .env.example .env
# Edit .env — replace all <generated> placeholders (see Environment Variables below)

# 2. Install dependencies
npm install

# 3. Start Supabase
docker compose up -d

# 4. Run migrations, data, and fixtures
task db:reset

# 5. Generate TypeScript types
task types:generate

# 6. Start dev server
task dev:next
```

## Environment Variables

All variables are in `.env`. The setup wizard generates secrets automatically. For manual setup, replace `<generated>` placeholders.

### Project

| Variable | Description | Default |
|----------|-------------|---------|
| `PROJECT_NAME` | Docker container prefix | `phpro-crm` |
| `NEXT_PUBLIC_APP_NAME` | Display name in the UI | `PHPro CRM` |
| `NEXT_PUBLIC_DEMO_MODE` | Enable demo mode banner | `true` |

### Ports

| Variable | Description | Default |
|----------|-------------|---------|
| `KONG_HTTP_PORT` | Supabase API gateway | `8000` |
| `POSTGRES_EXPOSED_PORT` | Postgres (host access) | `5432` |
| `STUDIO_PORT` | Supabase Studio dashboard | `3001` |

### Supabase URLs

| Variable | Description | Default |
|----------|-------------|---------|
| `SUPABASE_URL` | Server-side Supabase URL | `http://localhost:8000` |
| `NEXT_PUBLIC_SUPABASE_URL` | Client-side Supabase URL | `http://localhost:8000` |
| `SITE_URL` | App URL (used by GoTrue for redirects) | `http://localhost:3000` |

### Secrets (auto-generated by setup wizard)

| Variable | Description |
|----------|-------------|
| `JWT_SECRET` | JWT signing key (shared by all Supabase services) |
| `ANON_KEY` | Supabase anonymous JWT (public, limited access) |
| `SERVICE_ROLE_KEY` | Supabase service role JWT (bypasses RLS) |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Same as `ANON_KEY`, exposed to the browser |
| `SUPABASE_SERVICE_ROLE_KEY` | Same as `SERVICE_ROLE_KEY`, server-only |
| `POSTGRES_PASSWORD` | Postgres password |
| `SECRET_KEY_BASE` | Realtime service encryption key |
| `WEBHOOK_SECRET` | Webhook signature verification key |
| `VAULT_ENC_KEY` | Supabase Vault encryption key |
| `PG_META_CRYPTO_KEY` | Postgres Meta API encryption key |

### Auth (GoTrue)

| Variable | Description | Default |
|----------|-------------|---------|
| `JWT_EXP` | JWT expiration in seconds | `3600` |
| `GOTRUE_EXTERNAL_EMAIL_ENABLED` | Enable email/password auth | `true` |
| `GOTRUE_MAILER_AUTOCONFIRM` | Auto-confirm email on signup | `true` |
| `GOTRUE_SMTP_HOST` | SMTP host for emails | `inbucket` |
| `GOTRUE_SMTP_PORT` | SMTP port | `2500` |

## Task Commands

Run `task --list` to see all available commands.

### Development

| Command | Description |
|---------|-------------|
| `task setup` | Run the full interactive project setup |
| `task dev` | Start Supabase and Next.js dev server (auto-setup on first run) |
| `task dev:supabase` | Start Supabase services via Docker Compose |
| `task dev:next` | Start Next.js development server |
| `task dev:stop` | Stop all Supabase Docker services |

### Database

| Command | Description |
|---------|-------------|
| `task db:reset` | Full rebuild: drop schema, run migrations, data, and fixtures |
| `task db:migrate` | Run all schema migration files |
| `task db:new-migration -- <name>` | Create a new numbered migration file |
| `task db:data` | Apply production data (safe for all environments) |
| `task db:fixtures` | Load demo fixtures (dev/staging only) |
| `task db:seed` | Run both data and fixtures |
| `task db:studio` | Open Supabase Studio in the browser |

### Code Quality

| Command | Description |
|---------|-------------|
| `task lint` | Run ESLint |
| `task typecheck` | Run TypeScript type checking |
| `task build` | Build the Next.js application |

### Code Generation

| Command | Description |
|---------|-------------|
| `task types:generate` | Generate TypeScript types from the database schema |
| `task entity:new -- <name>` | Scaffold a new entity (migration, types, queries, actions, components) |

### Production & Infrastructure

| Command | Description |
|---------|-------------|
| `task prod:up` | Start production stack with Docker Compose |
| `task clean` | Stop and remove all containers and volumes |
| `task logs` | Tail all Docker Compose logs |
| `task logs:supabase -- <service>` | Tail logs for a specific service (e.g. `auth`, `rest`, `db`) |

## Database Architecture

```
supabase/
  migrations/    # Schema only (DDL, triggers, RLS, grants)
  data/          # Production data (pipelines, indices, settings)
  fixtures/      # Demo data (fake users, sample accounts)
  seed.sql       # Orchestrator for supabase db reset
```

### Deploy pipeline

```
production:   task db:migrate → task db:data
staging:      task db:migrate → task db:data → task db:fixtures
local dev:    task db:reset
```
